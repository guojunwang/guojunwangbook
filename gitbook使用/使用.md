## 标题
语法输入

```
# H1 一级标题
## H2 二级标题
### H3 三级标题
#### H4 四级标题
##### H5 五级标题
###### H6 六级标题

```
语法对应输出

# H1 一级标题
## H2 二级标题
### H3 三级标题
#### H4 四级标题
##### H5 五级标题
###### H6 六级标题




## 强调

强调：又叫做斜体，使用2个星号或2个下划线包着左右各一个

重点强调:又叫做粗体，使用4个星号或4个下划线包着左右各一个

删除线使用两个波浪线包着，左右各一个

语法输入

```
*我是斜体*  或  _我是斜体_
**我是粗体** 或 __我是粗体__
~~我是删除线~~
```

语法对应输出


*我是斜体*  或  _我是斜体_  
**我是粗体** 或 __我是粗体__  
~~我是删除线~~

## 列表

语法输入

```
1. 有序列表的第一项
2. 另外一个项
    * 无序子列表
3. 事实上序号不起作用，那只是一个数字而已
    1. 有序子列表
4. 最后一个项

* 无序列表可以使用星号
- 或者减号
+ 或者加号

```
语法对应输出


1. 有序列表的第一项
2. 另外一个项
    * 无序子列表
1. 事实上序号不起作用，那只是一个数字而已
    1. 有序子列表
4. 最后一个项


* 无序列表可以使用星号
- 或者减号
+ 或者加号

## 背景 
给文字添加背景只需要在文字的两边添加反引号 `即可 

语法输入


```
`我是文字背景`
```
语法对应输出

`我是文字背景`

##代码和语法高亮

`代码` 由含三个`反引号` ``` 的行包围，想支持那种语言代码高亮反引号后面名字就写对应的语言名称，如果不指明语言，代码就不会有高亮显示

语法输入


\```javascript  
var s = "JavaScript语法高亮";  
alert(s);  
\```


\```python  
s = "Python语法高亮"  
print s  
\```


\```  
var s = "JavaScript语法高亮";  
\```



语法对应输出

```javascript
var s = "JavaScript语法高亮";
alert(s);
```

```python
s = "Python语法高亮"
print s
```

```
#没有指明语言，所有没有语法高亮。
var s = "JavaScript语法高亮";

```
##表格

冒号可以用来对其列,连字符分隔符号`-`至少三个，外部的管道符 (|) 管道是可选的
* ---:为右对齐
* :---为左对齐
* :---:为居中对齐
* ---为默认左对齐

语法输入

```
| 左对齐 | 居中对齐| 右对齐|
|:------ |:-----:| -----:|
| 果郡王 | 果郡王 | 果郡王 |
| 果郡王 | 果郡王 | 果郡王 |
| 果郡王 | 果郡王 | 果郡王 |

```
语法对应输出

| 左对齐 | 居中对齐| 右对齐|
|:------ |:-----:| -----:|
| 果郡王 | 果郡王 | 果郡王 |
| 果郡王 | 果郡王 | 果郡王 |
| 果郡王 | 果郡王 | 果郡王 |


你还可以在表格中内嵌其他Markdown。

Markdown | Less | Pretty
-------- | ---- | ---
*Still*  | `renders` | **nicely**
1        | 2         | 3

## 反斜杠(忽略Markdown语法)

反斜杠使用\表示反斜杠，相当于反转义作用。在你不想显示Markdown标记时可以使用反斜杠。如一级标题
 
\# 我是反斜杠忽略的一级标题

## 换行符

单行换行，在需要换行的位置空格键敲击它两次，按下enter键即可换行  
我是单行换行，我是空格键敲击它两次，按下enter键后的效果

段落换行，我是单独的一个段落，我是被按下两次enter键后的效果

## 脚注

Markdown默认使用的脚注样式链接不会在页面中显示。有时包含一个读者可见的非超链接注脚很有用。对于这样的注脚，GitBook支持的一种简单的语法。
Text prior to footnote reference.[^2]
[^2]: Comment to include in footnote.

## 内嵌HTML


你同样可以在Markdown中使用HTML，并且它能很好的工作。
<dl>
  <dt>Definition list</dt>
  <dd>Is something people use sometimes.</dd>

  <dt>Markdown in HTML</dt>
  <dd>Does *not* work **very** well. Use HTML <em>tags</em>.</dd>
</dl>
